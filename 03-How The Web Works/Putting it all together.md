## 📝 Task 1: Putting It All Together

### Website Request Flow

When a user enters a website URL into a browser:

1. The browser requests the website.
2. DNS translates the domain name into an IP address.
3. The browser connects to the web server.
4. The server responds using the HTTP protocol.
5. The browser renders HTML, CSS, JavaScript, images, and other resources.
6. The website becomes visible to the user.

### Key Concepts

* **DNS (Domain Name System):** Converts domain names into IP addresses.
* **HTTP/HTTPS:** Protocol used for communication between client and server.
* **Web Server:** Delivers website content to users.
* **Browser:** Displays the received content.

## 📝 Task 2: Other Components

### Load Balancers

Load balancers distribute incoming traffic across multiple servers.

#### Benefits:

* Improved performance
* High availability
* Fault tolerance
* Health monitoring of servers

#### Algorithms:

* Round Robin
* Weighted Distribution

---

### CDN (Content Delivery Network)

A CDN stores website resources on servers around the world.

#### Benefits:

* Faster content delivery
* Reduced latency
* Lower server load
* Improved user experience

Examples of cached resources:

* Images
* CSS
* JavaScript
* Videos

---

### Databases

Databases store and manage website information.

Examples:

* MySQL
* PostgreSQL
* Microsoft SQL Server
* MongoDB

Common uses:

* User accounts
* Login information
* Website content
* Application data

---

### WAF (Web Application Firewall)

A WAF sits between users and web servers.

#### Functions:

* Blocks malicious requests
* Protects against common web attacks
* Performs rate limiting
* Detects suspicious traffic

---

## 📝 Task 3: How Web Servers Work

### What is a Web Server?

A web server listens for incoming requests and serves web content using HTTP.

Popular web servers:

* Apache
* Nginx
* IIS
* Node.js

---

### Virtual Hosts

Virtual Hosts allow a single web server to host multiple websites.

Example:

website1.com → /var/www/website1

website2.com → /var/www/website2

This enables multiple domains to run on the same server.

---

### Static vs Dynamic Content

#### Static Content

Content that does not change.

Examples:

* Images
* CSS files
* JavaScript files
* Static HTML pages

#### Dynamic Content

Content generated based on user requests.

Examples:

* Search results
* Blogs
* User dashboards
* E-commerce websites

---

### Backend Languages

Backend languages process requests and generate dynamic content.

Examples:

* PHP
* Python
* Ruby
* Node.js
* Perl

The backend communicates with databases and returns processed information to the frontend.
